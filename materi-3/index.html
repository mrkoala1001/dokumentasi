Evolusi Pengiriman Halaman

Perbedaan mendasar antara Multi-Page Application (MPA) dan Single Page Application (SPA) terletak pada cara browser berkomunikasi dengan server setelah halaman pertama dimuat. Diagram di bawah ini mengilustrasikan alur proses tersebut.

📄 MPA (Tradisional)

Setiap navigasi memicu pemuatan ulang halaman HTML secara utuh dari server.

💻 Klien (Klik Link) --- Permintaan ---→ 🗄️ Server
💻 Klien (Render Ulang) ←--- HTML Penuh --- 🗄️ Server (Proses HTML)

⚡ SPA (Modern)

Satu halaman HTML dimuat di awal. Navigasi selanjutnya hanya meminta data (JSON).

💻 Klien (Klik Menu) --- Permintaan ---→ 🗄️ Server
💻 Klien (Update UI Dinamis) ←--- Data JSON --- 🗄️ Server (Kirim Data)

Analisis Fitur Utama

Grafik radar ini membandingkan MPA, SPA, dan PWA pada lima dimensi krusial. MPA unggul mutlak di SEO, sementara SPA merajai kecepatan interaksi (UX). PWA hadir sebagai jembatan yang memberikan kapabilitas offline dan pengalaman menyerupai aplikasi native, mengangkat skor SPA ke level berikutnya.

Metrik Kecepatan (Ilustratif)

Visualisasi ini menunjukkan kontras performa. SPA memiliki waktu beban awal (Initial Load) yang sedikit lebih tinggi karena harus mengunduh bundle JavaScript besar. Namun, perhatikan waktu navigasi selanjutnya: SPA dan SSR/SSG (melalui teknik hidrasi) jauh lebih cepat dibandingkan MPA yang harus merender ulang HTML secara penuh.
